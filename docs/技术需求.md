# 游戏网站仿站技术需求文档

## 技术架构
- **前端框架**: Nuxt.js 3
- **样式框架**: TailwindCSS
- **部署方式**: 混合渲染 (SSG + CSR)
- **主要依赖库**:
  - `@nuxtjs/tailwindcss`: TailwindCSS集成.
  - `@headlessui/vue`: 提供无样式的、完全可访问的UI组件.
  - `@heroicons/vue`: 精美的SVG图标库.
  - `puppeteer`: 用于自动化生成游戏缩略图的无头浏览器工具.

```javascript
export const useGames = () => {
  const { $fetch } = useNuxtApp()
  
  // 获取所有游戏
  const getAllGames = async () => {
    const { games } = await $fetch('/data/games.json')
    return games
  }
  
  // 按分类获取游戏
  const getGamesByCategory = async (category) => {
    const games = await getAllGames()
    return games.filter(game => game.category === category)
  }
  
  // 获取热门游戏
  const getHotGames = async () => {
    const games = await getAllGames()
    return games.filter(game => game.isHot)
  }
  
  // 搜索游戏
  const searchGames = async (keyword) => {
    const games = await getAllGames()
    return games.filter(game => 
      game.title.toLowerCase().includes(keyword.toLowerCase()) ||
      game.description.toLowerCase().includes(keyword.toLowerCase())
    )
  }
  
  return {
    getAllGames,
    getGamesByCategory,
    getHotGames,
    searchGames
  }
}
```

## 游戏来源管理

项目采用**混合游戏集成模式**，支持"外部`iframe`嵌入"和"本地文件托管"两种方式，以实现最大的灵活性。

### 1. 游戏集成方案

#### ✅ 主要方案: 外部`iframe`嵌入
对于来自 itch.io、CodePen 等平台的外部游戏，直接通过 `iframe` 嵌入。

- **优点**: 实施简单，不占用项目存储空间，便于集成大量游戏。
- **配置**: 在 `games.json` 中，设置 `gameType: "iframe"` 并提供完整的 `gameUrl`。

#### ✅ 辅助方案: 本地文件托管
对于开源或已获得授权的HTML5游戏，可以将其文件直接托管在项目中。

- **优点**: 加载速度快，无外部依赖，可离线访问。
- **目录**: 游戏文件存放于 `public/gfiles/` 目录下，每个游戏一个子目录。
- **配置**: 在 `games.json` 中，设置 `gameType: "local"`, `gameUrl` 指向游戏入口HTML文件（如 `/gfiles/my-game/index.html`）。

### 2. 自动化工具：缩略图生成
为了简化本地托管游戏的管理，项目提供了一个自动化脚本，用于批量生成游戏缩略图。

- **脚本路径**: `scripts/generate-thumbnails.cjs`
- **核心功能**: 使用 `puppeteer` 启动一个无头浏览器，访问本地开发服务器上的游戏地址，截取游戏首屏画面，并保存为缩略图。同时，脚本会自动更新 `games.json` 中对应游戏的 `thumbnail` 字段。
- **使用方法**:
  1. 启动本地开发服务器 (`npm run dev`)。
  2. 运行 `npm run generate-thumbnails` 命令。
- **注意**: 此脚本只会处理 `gameUrl` 以 `/gfiles/` 开头的本地游戏。

### 3. Data文件夹驱动的游戏管理
无论是哪种集成方式，所有游戏的核心信息都通过 `data/games.json` 文件进行统一管理。

## Cloudflare Pages 部署兼容性

### 1. 部署与渲染策略
项目采用**混合渲染模式**进行部署，以平衡SEO、性能和灵活性。

- **静态站点生成 (SSG)**: 网站的骨架页面（如首页、关于页）在构建时被预渲染为静态HTML，以实现最佳的加载性能和SEO效果。
- **客户端渲染 (CSR)**: 动态页面（如游戏详情页 `/game/[slug]` 和分类页 `/category/[slug]`）则在客户端进行渲染。这意味着浏览器会先加载一个基础的静态页面，然后通过JavaScript获取数据并动态生成页面内容。

⚠️ **注意事项**:
- **渲染模式**: 由于采用了CSR，动态页面的首屏加载速度可能略慢于纯SSG页面，但换来了更大的灵活性，无需在每次增删游戏时都重新构建所有页面。
- **大型游戏文件**: 超过Cloudflare Pages单文件25MB限制的游戏，需要托管在外部存储（如R2）并通过`iframe`引入。

### 2. 部署配置优化

#### nuxt.config.ts 配置
配置文件需要反映出混合渲染的策略。关键在于 `generate.routes` 保持为空，以关闭对动态路由的预渲染。

```javascript
export default defineNuxtConfig({
  // Cloudflare Pages 静态生成配置
  nitro: {
    preset: 'cloudflare-pages',
    prerender: {
      routes: ['/sitemap.xml'] // 只预渲染站点地图等少量关键静态路由
    }
  },
  
  // 静态站点生成配置
  generate: {
    // 保持为空，以关闭动态路由的预渲染，启用CSR模式
    routes: [] 
  },

  // ... 其他配置
})
```

#### _redirects 文件配置
为了让Cloudflare Pages能正确处理CSR路由，需要一个 `_redirects` 文件将所有未匹配到的路径重定向到 `index.html`，交由Vue Router处理。

```bash

```

## 数据维护流程

### 1. 添加新游戏
1. **对于本地游戏**: 将游戏文件放入 `public/gfiles/` 目录。
2. 在 `data/games.json` 中添加游戏的核心信息（`id`, `slug`, `title`等）。
3. **对于本地游戏**: 运行 `npm run generate-thumbnails` 自动生成缩略图并更新JSON文件。
4. **对于外部游戏**: 在 `data/games.json` 中填写完整信息，并手动添加 `thumbnail` 图片路径。
5. 提交代码变更，Cloudflare Pages会自动触发新的构建和部署。

### 2. 更新游戏分类
1. 修改 `data/categories.json`
2. 更新相关游戏的分类字段
3. 重新构建和部署站点（通常是提交代码后自动完成）。

### 3. 网站配置更新
- 修改 `data/site-config.json`。
- 提交代码变更以触发重新部署。
